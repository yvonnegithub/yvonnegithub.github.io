npm run & npm run-script

npm run 是 npm run-script 的缩写，一般都使用 前者，但是后者可以更好地反应这个命令的本质,它们都能执行 package.json 文件 scripts 字段下指定的任务。直接运行 npm run 或 npm run-script，不加参数会列出 scripts 属性下所有可运行的命令极其命令内容。

1.关于路径

npm run 命令会自动在环境变量 $PATH 添加 node_modules/.bin 目录，所以 scripts 字段里面调用命令时不用加上路径。所以我们在 package.json 文件内的 scripts 字段内指定任务的时候 一般 无需指定脚本文件的路径，只需要将脚本放到 ./node_module/.bin/ 目录下即可，命令会在 这个目录 下自动寻找对应的脚本文件。而无需使用 ./node_modules/.bin/jshint **.js。但是也是可以自己更改与指定你需要运行的脚本的文件位置

2.管道式命令

同时执行多个任务，可以借用 Linux 系统的 管道命令，将两个操作连在一起，在 package.json 文件的 scripts 字段内的一个任务可以由多个子任务组成，连接方式如下：

  1. |：连接两个任务
  2. &&：任务内部引用其他任务，子任务 先后 执行
  3. &：任务内部引用其他任务，子任务 平行 执行

3.bash 脚本

写在 scripts 属性中的命令，也可以在 node_modules/.bin 目录中直接写成 bash 脚本
只需要在命令行中执行npm run 脚本名就可以执行脚本

4.常见脚本命令

  1. start ：一般用于项目的初始化，是接下来一切工作的依赖起始端
  2. dev ：规定开发阶段所要做的处理，比如监视文件变化、实时编译
  3. server ：一般用于启动服务
  4. test ：一般用于执行测试
  5. prod ：一般用于规定进入生产环境时需要做的处理
  6. help ：展示帮助信息
  7. docs ：生成文档

5.pre- 和 post- 脚本命令

npm run 为每条命令提供了 pre- 和 post- 两个钩子。以 npm run test 为例，如果我们的 scripts 字段规定了 pretest 和 posttest.

则会先执行 pretest 任务，再执行 test 任务，完成 test 任务后即执行 posttest 任务。

node－npm

发表于 2016-10-07

导语：

Node.js是让Javascript脱离浏览器运行在服务器的一个平台，不是语言。实际上它是对Google V8引擎进行了封装。采用单线程、异步IO与事件驱动的设计来实现高并发（异步事件也在一定程度上增加了开发和调试的难度）；Node.js对一些特殊用例进行了优化，提供了替代的API，使得V8在非浏览器环境下运行得更好。它允许在后端（脱离浏览器环境）运行JavaScript代码。因此，Node.js事实上既是一个运行时环境，同时又是一个库。

一：关于npm

npm是随同NodeJS一起安装的包管理工具,新版本的node已经集成了npm，可用npm －v 命令查看是否已经安装。常见用法：

  1. 从NPM服务器下载别人编写的第三方包到本地使用
  2. 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用
  3. 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用

二：npm常用命令

1.使用 npm 命令安装模块

  • npm install moduleNames：安装Node模块,安装完毕后会在工程目录下产生一个node_modules目录，其目录下就是安装的各个node模块，在代码中代码中只需要通过 require(‘moduleNames’) 的方式就好，无需指定第三方包路径。

2.全局安装与本地安装

npm 的包安装分为本地安装（local）、全局安装（global）两种，从敲的命令行来看，差别只是有没有-g而已

  1. npm install moduleNames
  2. npm install -g moduleNames
本地安装：
  1. 将安装包放在 ./node_modules 下（运行 npm 命令时所在的目录），如果没有 node_modules 目录，会在当前执行 npm 命令的目录下生成 node_modules 目录。
  2. 可以通过 require() 来引入本地安装的包。
全局安装
  1. 将安装包放在 /usr/local 下或者你 node 的安装目录。
  2. 可以直接在命令行里使用。

3.卸载模块

  1. npm uninstall moduleNames

4.更新模块

  1. npm update moduleNames

5.创建模块

创建模块，package.json 文件是必不可少的。我们可以使用 NPM 生成 package.json 文件，生成的文件包含了基本的结果。

  1. npm init: 创建package.json文件
  2. npm adduser: 在 npm 资源库中注册用户
  3. npm npm publish: 发布模块

成功的话创建的模块就可以跟其它模块一样使用npm来安装

三：关于package.json

package.json位于模块的目录下，用来定义包的属性

属性说明

  • name - 包名
  • version - 包的版本号
  • description - 包的描述
  • homepage - 包的官网 url
  • author - 包的作者姓名
  • contributors - 包的其他贡献者姓名。
  • dependencies - 依赖包列表。如果依赖包没有安装，npm 会自动将依赖包安装在 node_module 目录下
  • repository - 包代码存放的地方的类型，可以是 git 或 svn，git 可在 Github 上。
  • main - main 字段是一个模块ID，它是一个指向你程序的主要项目。就是说，如果你包的名字叫 express，然后用户安装它，然后require(“express”)。
  • keywords - 关键字

使用淘宝npm代替官方版本

npm install -g cnpm –registry=https://registry.npm.taobao.org

安装成功之后可使用cnpm代替npm进行安装

伪类与伪元素

发表于 2016-09-27

导语

css可以通过id，class，属性选取元素，还可以通过元素的特殊状态去选取元素，这就是伪类与伪元素。伪类与伪元素是独立于文档元素，预定义的,不同于id选择器，类选择器等直接从html文档层次结构（就是DOM树）选取元素。伪类和伪元素使用“：”与其它选择器区分.

一. 伪类

通过伪类是通过元素当前所处的状态（或者特性）去选择元素，也就是说当元素达到某个状态的时候就会获得这个样式，当它不再处于这个状态时，它又会失去这个样式，它和类的功能类似，但是独立于文档之外.

常用的伪类：

  • :link,应用于未访问过的链接
  • :hover,鼠标悬停
  • :active,用于被激活的元素：被惦记的链接／按下的按钮
  • :visited,用于被访问过的链接
  • :lang,用于带指定lang的元素
  • :focus,用于获得键盘输入焦点的元素
  • :first-child
注意：对于同一个元素，:hover必须位于:link，visitied之后才会生效，:active必须位于:hover之后才会生效

二. 伪元素

伪元素针对元素中特定的内容，比如选区某个元素的第一个字，第一行，选取内容前面或者后面的内容，控制的实际上也是元素，但是本身不存在于文档中.

1.常用的伪元素：

  • :first-letter,用于元素文本的第一个字母
  • :first-line,用于元素的第一行
  • :before,在元素的最前面添加内容
  • :after,在元素的最后面添加内容

2.关于：before，：after的用法

content属性（必须): 用于在元素的头部的或者尾部添加内容，添加的内容不会出现在DOM中,不可复制，仅仅是在css渲染层加入；并且不能为空.

3.使用伪元素清除浮动:

1.原本：添加一个空的div，应用clear：both属性


.clear-fix{overflow:hidden;zoom:1}
2.伪元素： 

.clear-fix:after{display:table,content:” ”,clear:both}
扩展：配合display：block，position：absolute可以将一个特定的元素当成两个容器拼合出现双边框的效果（例子待补充).

4.补充关于:和::的问题

仅仅需要使用这两个伪元素添加一些简单的字符的时候，只需要使用伪类的单冒号写法，以保证浏览器的兼容性；在css3中为了区别伪类与伪元素为伪元素使用了双冒号，因此如果使用了display和width等使元素脱离了原本的元素，建议按照标准双写.
