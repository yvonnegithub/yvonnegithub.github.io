npm run & npm run-script

npm run 是 npm run-script 的缩写，一般都使用 前者，但是后者可以更好地反应这个命令的本质,它们都能执行 package.json 文件 scripts 字段下指定的任务。直接运行 npm run 或 npm run-script，不加参数会列出 scripts 属性下所有可运行的命令极其命令内容。

1.关于路径

npm run 命令会自动在环境变量 $PATH 添加 node_modules/.bin 目录，所以 scripts 字段里面调用命令时不用加上路径。所以我们在 package.json 文件内的 scripts 字段内指定任务的时候 一般 无需指定脚本文件的路径，只需要将脚本放到 ./node_module/.bin/ 目录下即可，命令会在 这个目录 下自动寻找对应的脚本文件。而无需使用 ./node_modules/.bin/jshint **.js。但是也是可以自己更改与指定你需要运行的脚本的文件位置

2.管道式命令

同时执行多个任务，可以借用 Linux 系统的 管道命令，将两个操作连在一起，在 package.json 文件的 scripts 字段内的一个任务可以由多个子任务组成，连接方式如下：

  1. |：连接两个任务
  2. &&：任务内部引用其他任务，子任务 先后 执行
  3. &：任务内部引用其他任务，子任务 平行 执行

3.bash 脚本

写在 scripts 属性中的命令，也可以在 node_modules/.bin 目录中直接写成 bash 脚本
只需要在命令行中执行npm run 脚本名就可以执行脚本

4.常见脚本命令

  1. start ：一般用于项目的初始化，是接下来一切工作的依赖起始端
  2. dev ：规定开发阶段所要做的处理，比如监视文件变化、实时编译
  3. server ：一般用于启动服务
  4. test ：一般用于执行测试
  5. prod ：一般用于规定进入生产环境时需要做的处理
  6. help ：展示帮助信息
  7. docs ：生成文档

5.pre- 和 post- 脚本命令

npm run 为每条命令提供了 pre- 和 post- 两个钩子。以 npm run test 为例，如果我们的 scripts 字段规定了 pretest 和 posttest.

则会先执行 pretest 任务，再执行 test 任务，完成 test 任务后即执行 posttest 任务。
